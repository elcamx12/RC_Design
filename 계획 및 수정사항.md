## 1. 계획

### 📄 구조계산서 PDF 연동 (구현 완료 — v0.7.0)

#### 현재 구현 상태

```
구조계산서 PDF
  ↓ 자동 분석 (위치 기반 파싱) 또는 AI 분석 (Claude Haiku + 이미지)
부재력 (Mu, Vu, Pu) + 단면 + 배근 + Loc + 설계강도
  ↓ 검토 모드 입력
구조계산서 배근 기반 φMn 계산 → OK/NG 판정
```

#### 추출 가능한 값 (MIDAS Gen / BeST.RC / BeST.Steel)

| 부재 | 추출 항목 | 비고 |
|------|-----------|------|
| **보** | Mu(-/+, I/MID/J), Vu(I/MID/J), φMn, φVc, φVs, Loc, 배근, 검토비 | MIDAS: 위치별 전수, BeST: 단일값 |
| **기둥** | Pu, Mux, Muy, Vu, 배근, P-M 검토비, 단면 | BeST.Steel 다중 페이지 지원 |

#### 검토 가능/불가 항목

| 항목 | 가능? | 설명 |
|------|:----:|------|
| 휨 검토 (φMn vs Mu) | ✅ | 구조계산서 배근 기반 φMn 계산 |
| 전단 검토 (φVc+φVs vs Vu) | ✅ | 스터럽 간격 기반 |
| 처짐 | ❌ | D/L 분리 모멘트 필요 (추후 1.4D 조합 역산으로 가능) |
| IMF/접합부 | ❌ | 집중하중으로 불가 |
| 단면 변경 | ❌ | 추후 확장 (검토만 가능) |

#### MIDAS vs BeST 병용 참고
- MIDAS Gen: 골조 해석 → 부재력 산출 + 설계 검토
- BeST.RC/Steel: 독립 단면설계 → MIDAS 교차 검증
- 동일 부재명이어도 **배근/Loc/설계강도가 다를 수 있음** → 분리 유지

### 📙 검토 모드 (구조계산서 기반)

#### 🔴 로직/코드 오류

47. **φMn과 MIDAS 약 6% 차이** — MIDAS는 D19=300mm²(공칭) + α₁=0.85(구기준). 우리는 286.5mm² + α₁=0.80(KDS 2021). 프로그램 간 기준 차이. #64(구조계산서 제공값 기반 계산)로 해결 가능
66. **보 h ≤ cover일 때 d 음수** — `_review_beam_location`에서 h가 cover보다 작으면 d가 음수되어 계산 오류. 입력 검증 추가 필요
67. **기둥 c_column ≤ 0일 때 ZeroDivisionError** — `_review_column`에서 Ag=0, rho 나눗셈 오류. 입력 검증 추가 필요

#### 🟠 기능 확장

51. **처짐 검토** — D/L 분리 시 가능 (1.4D 조합 역산)
52. **사용자 단면 변경 → 재검토** — 현재 검토만, 단면 수정 기능 추후
53. **자동 최적화 제안** — 검토 NG 시 적정 배근 제안 기능 (장기, 설계 모드 로직 참고)
64. **구조계산서 제공값 기반 계산** — 검토 모드 철학: 구조계산서에서 β₁ 등 값이 제공되면 그 값을 그대로 사용하여 계산. BeST PDF에서 β₁=0.800 추출되면 그대로 사용, MIDAS처럼 미제공이면 KDS 표로 계산. 파서에서 β₁/α₁/cb 등 추출 → 입력 → 계산에 반영하는 구조 필요.
65. **SRC 합성기둥 엔진** — 현재 RC 전용. BeST.Steel은 SRC(강재+RC) 합성단면이므로 ΦPn/ΦMn이 ~25% 차이. 강재(Steel Section) 기여를 P-M 계산에 반영 필요 (Po = As·Fy,Stl + Asr·Fy,Bar + 0.85Ac·fck). 전단도 SRC 공식 필요.
