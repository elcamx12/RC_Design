# 시스템 구조도 사용 매뉴얼

## 1. 개요

`diagram.html`은 **Cytoscape.js 기반 인터랙티브 아키텍처 다이어그램**입니다. 브라우저에서 열면 노드(박스/함수)와 엣지(화살표)로 구성된 시스템 구조도가 표시됩니다. preview 서버로 실행하며, `preview_eval`로 실시간 조작이 가능합니다.

## 2. 핵심 구조

### 노드 생성 — `n()` 함수
```javascript
n(id, label, parent, category, x, y)
```
- `id`: 고유 식별자 (예: `'EB_FLEX'`)
- `label`: 표시 텍스트 (예: `'flexural_design()'`)
- `parent`: 부모 노드 ID (그룹에 포함시킬 때, 없으면 `null`)
- `category`: 스타일 카테고리 (`'entry'`, `'input'`, `'calc'`, `'engine'`, `'merged'`, `'decision'`, `'result'`)
- `x, y`: 초기 좌표

### 엣지 생성 — `e()` 함수
```javascript
e(source, target, label, category)
```
- `source → target` 방향 화살표
- `label`: 화살표 위에 표시되는 설명
- `category`: `'flow'`(기본 회색), `'data'`(주황 점선), `'return'`(초록 점선), `'bidir'`(양방향), `'cross'`(파란 점선)

## 3. 카테고리별 색상

| 카테고리 | 색상 | 용도 |
|---|---|---|
| `entry` | 보라색 | 진입점 (app.py, start.py) |
| `input` | 초록색 | 입력부 (UI, PDF 파싱) |
| `calc` | 빨간색 | 계산 로직 (calc_manager) |
| `engine` | 갈색 | 엔진 함수 (beam/column/slab) |
| `merged` | 주황색 | 계산+엔진 병합 노드 |
| `decision` | 주황 다이아몬드 | 분기 판정 (OK/NG) |
| `result` | 청록색 | 출력/시각화/결과 |

## 4. 설명(desc) 시스템

각 노드에는 `desc` 데이터가 있어 **DOM 오버레이**로 노드 아래에 흰색 텍스트로 표시됩니다. 함수명은 카테고리 색상, 설명은 항상 흰색입니다.

```javascript
// 설명 추가/수정
cy.getElementById('EB_FLEX').data('desc', '새로운 설명 텍스트');
updateDescPositions(); // 오버레이 위치 갱신
```

## 5. 주요 기능

| 기능 | 조작 |
|---|---|
| 확대/축소 | 마우스 휠 (마우스 위치 기준) |
| 화면 이동 | 마우스 드래그 |
| 노드 이동 | 노드 클릭+드래그 (25px 그리드 스냅) |
| 박스 접기/펼치기 | 우클릭 |
| 전체 보기 | 상단 '전체 보기' 버튼 |
| 위치 초기화 | 상단 '위치 초기화' 버튼 |
| 위치 저장 | localStorage에 자동 저장 |

## 6. 실시간 조작 (preview_eval)

### 노드 추가
```javascript
cy.add({
  data: { id: 'NEW_NODE', label: '새 함수()', cat: 'calc', parent: 'PARENT_ID', desc: '설명' },
  position: { x: 500, y: 300 }
});
updateDescPositions();
```

### 엣지 추가
```javascript
cy.add({
  data: { source: 'A', target: 'B', label: '호출', cat: 'flow' }
});
```

### 노드 위치 변경
```javascript
cy.getElementById('NODE_ID').position({ x: 100, y: 200 });
savePositions();
updateDescPositions();
```

### 설명 일괄 수정
```javascript
var descs = { 'NODE1': '설명1', 'NODE2': '설명2' };
Object.keys(descs).forEach(function(id) {
  cy.getElementById(id).data('desc', descs[id]);
});
updateDescPositions();
```

### 현재 좌표 추출
```javascript
(function(){
  var r = {};
  cy.nodes().forEach(function(n) {
    var p = n.position();
    r[n.id()] = { x: Math.round(p.x), y: Math.round(p.y) };
  });
  return JSON.stringify(r);
})()
```

## 7. 위치 영구 저장 구조

- **localStorage**: 사용자가 드래그한 위치 실시간 저장 (브라우저별)
- **OPTIMAL 맵**: `diagram.html` 코드 내 하드코딩된 기본 좌표. localStorage가 없을 때 사용
- **n() 좌표**: OPTIMAL에도 없을 때의 최종 폴백

우선순위: `localStorage > OPTIMAL > n() preset`

## 8. 코드에 영구 반영하기

preview_eval로 조작한 내용은 브라우저 메모리에만 있으므로, 영구 저장하려면:

1. **노드/엣지 추가**: `diagram.html`의 `var nodes = [...]`, `var edges = [...]` 배열에 직접 추가
2. **좌표 저장**: OPTIMAL 맵에 좌표 추가
3. **설명 저장**: `NODE_DESCS` 객체에 추가

## 9. 새 프로젝트에 적용하기

1. `diagram.html`을 복사
2. `var nodes = [...]` 배열을 새 프로젝트 구조로 교체
3. `var edges = [...]` 배열을 새 연결로 교체
4. `NODE_DESCS` 객체를 새 설명으로 교체
5. `OPTIMAL` 맵 비우기 (배치 후 다시 채움)
6. 제목 변경 (`<title>`, `<h1>`)

## 10. 주요 JS 함수 목록

| 함수 | 용도 |
|---|---|
| `savePositions()` | 현재 위치 → localStorage |
| `loadPositions()` | localStorage → 노드 위치 복원 |
| `updateDescPositions()` | 설명 오버레이 위치 갱신 |
| `fitAll()` | 전체 노드가 보이게 줌 |
| `resetPositions()` | OPTIMAL 기본 위치로 초기화 |
| `collapseNode(id)` | 박스 접기 |
| `expandNode(id)` | 박스 펼치기 |
